1. INTRODUCTION                                             *clide-introduction*

clide.nvim implements the Claude Code IDE protocol (WebSocket-MCP) in pure
Lua. Edits proposed by Claude render as inline hunks in your buffers with
per-hunk accept/reject. An SSE MCP server exposes all 17 IDE tools to
Claude Code via the standard MCP transport. Multiple Claude CLI clients
connect simultaneously to one WS server, each with independent RPC dispatch.

==============================================================================
2. REQUIREMENTS                                             *clide-requirements*

- Neovim >= 0.10
- `claude` CLI in PATH
- nvim-lua/plenary.nvim

==============================================================================
3. CONFIGURATION                                           *clide-configuration*

Defaults shown. All keys optional — |clide-setup| works without arguments.

>lua
  require("clide").setup({
    autostart = false,
    sse_port = 42069,
    follow = false,
    auto_install_mcp = true,
    log_level = "info",
    terminal = {
      provider = "auto",
      cmd = "claude",
      split_side = "right",
      split_width = 0.35,
    },
    review = {
      inline = true,
      hint_line = true,
      keymaps = {
        accept = "<Leader>ma",
        reject = "<Leader>mr",
        accept_all = "<Leader>mA",
        reject_all = "<Leader>mR",
        next_hunk = "]h",
        prev_hunk = "[h",
      },
    },
    cmd_keymaps = {
      toggle = "<Leader>mt",
      start = "<Leader>ms",
      stop = "<Leader>mq",
      log = "<Leader>ml",
      send = "<Leader>me",
      send_toggle = "<Leader>mz",
    },
  })
<

terminal.provider: "auto" | "tmux" | "toggleterm" | "snacks" | "native" | "none"
  auto resolves: tmux ($TMUX) → snacks (if installed) → native

review.inline: true = inline hunks | false = side-by-side diff tab

Set any cmd_keymaps entry to false to disable that mapping.

5. COMMANDS                                                     *clide-commands*

:ClideStart          Start server, write lock file, launch claude.
:ClideStop           Stop server, remove lock file.
:ClideRestart        Stop then start.
:ClideToggle         Toggle the claude terminal (starts if stopped).
:{range}ClideSend    At-mention the range in Claude's context.
:ClideReviewTab      Reopen the current inline review as a diff tab.
:ClideInstallHooks   Install status hooks into .claude/settings.local.json.
:ClideInstallMCP     Write MCP server config to .mcp.json and auto-approve
                     in .claude/settings.local.json (auto-run on :ClideStart
                     when auto_install_mcp = true, the default).
:ClideLog            Show the log ring buffer.
:checkhealth clide   Diagnose setup: Neovim, claude CLI, plenary, providers.

==============================================================================
6. REVIEW KEYMAPS                                                 *clide-review*

Active while a review has pending hunks (buffer-local, configurable):

  <Leader>ma    accept hunk under/near cursor
  <Leader>mr    reject hunk
  <Leader>mA    accept all hunks in buffer
  <Leader>mR    reject all hunks in buffer
  ]h           next pending hunk (crosses files)
  [h           previous pending hunk

Global (normal mode):

  <Leader>ms   :ClideStart
  <Leader>mq   :ClideStop
  <Leader>mt   :ClideToggle
  <Leader>ml   :ClideLog

==============================================================================
7. TERMINAL PROVIDERS                                          *clide-providers*

auto (default) resolves: tmux (inside $TMUX) > snacks.nvim (if installed)
> native |:terminal|. Set terminal.provider to force one, or "none" to run
claude yourself with the env vars clide prints.

==============================================================================
8. STATUSLINE                                                 *clide-statusline*

>lua
  require("clide.status").lualine
<

States: working, waiting, idle, disconnected — driven by Claude Code hooks
(run :ClideInstallHooks once per project). Includes pending review count.

==============================================================================
9. FAQ                                                                 *clide-faq*

Claude doesn't connect?
  Run |:checkhealth| clide. Ensure `claude` CLI is in PATH and :ClideStart
  ran without errors.

"E444: Cannot close last window"?
  The claude terminal is the only window. Open a file or :ClideToggle.

Multiple Claude instances?
  Yes. Each connects as its own session — side-by-side agents on different
  files.

Hunks not showing?
  Check review.inline = true (default). Use :ClideReviewTab for side-by-side
  diff fallback.

Stop everything?
  :ClideStop — kills server, removes lockfile, closes all sessions.

Port conflict?
  Set sse_port in |clide-configuration| to a different value.
