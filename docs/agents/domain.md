# Domain Docs

How engineering skills consume this repo's domain documentation.

## Before exploring, read these

- **`openwiki/quickstart.md`** — project overview, architecture, workflow
- **`docs/ROADMAP.md`** — known issues, gaps, design decisions
- **`openwiki/architecture.md`** — transport, notification flow, design notes
- **`CLAUDE.md`** — hard constraints, layout, workflow rules

## Layout

Single-context repo. No monorepo split.

## Use glossary vocabulary

When output names a domain concept, use the term as defined in openwiki or CLAUDE.md.
Don't drift to synonyms the repo avoids.
